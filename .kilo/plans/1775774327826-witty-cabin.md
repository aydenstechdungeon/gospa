## 1. Security Vulnerabilities & Exploits

### 1.1 Critical: Weak CSP in `form-remote/` Example

**File:** `examples/form-remote/main.go:18`
```go
config.ContentSecurityPolicy = "default-src 'self'; script-src 'self' 'unsafe-inline' https://unpkg.com unpkg.com; ..."
```

**Issues:**
- `'unsafe-inline'` in `script-src` and `style-src` defeats the purpose of CSP
- Bare `unpkg.com` (no `https://` prefix) is protocol-relative and could allow HTTP injection
- No nonce-based CSP despite framework supporting it

**PoC:**
```bash
# An attacker with XSS can inject arbitrary scripts:
# <script>alert(document.cookie)</script>
# Would execute because 'unsafe-inline' allows it
```

**Mitigation:**
```go
// Use nonce-based CSP instead:
config.ContentSecurityPolicy = "default-src 'self'; script-src 'self' 'nonce-{nonce}' https://unpkg.com; style-src 'self' 'nonce-{nonce}' https://unpkg.com; img-src 'self' data: https:; connect-src 'self' wss: https:;"
```

**Severity:** Critical

---

### 1.2 Critical: `GOSPA_WS_INSECURE=1` Overrides Production Safety

**Files:** `gospa.go:255`, `config.go:244`
```go
// gospa.go:255
if os.Getenv("GOSPA_WS_INSECURE") == "1" {
    config.AllowInsecureWS = true
}
```

**Issue:** Any environment variable leak, CI artifact, or accidental deployment with `GOSPA_WS_INSECURE=1` forces insecure `ws://` even on HTTPS production pages. This is a mixed-content vulnerability.

**PoC:**
```bash
# Deploy with this env var set accidentally:
GOSPA_WS_INSECURE=1 ./server
# All WebSocket traffic goes over unencrypted ws:// even on https://
# Enables MITM interception of all real-time state
```

**Mitigation:** Block `GOSPA_WS_INSECURE` when `DevMode: false` or add a second explicit override like `GOSPA_FORCE_INSECURE_WS=I_UNDERSTAND_THE_RISK`.

**Severity:** Critical

---

### 1.3 High: No Security Headers on Website

**File:** `website/main.go`

**Issue:** The documentation website at `gospa.onrender.com` has no:

- `X-Frame-Options` (clickjacking risk)

- `Referrer-Policy`
- `Permissions-Policy`

**Mitigation:** Add security header middleware to `website/main.go` before route registration.

**Severity:** High

### 1.4 Medium: CDN Dependency Without SRI in Examples

**Files:** `examples/counter/routes/layout.templ`, `examples/form-remote/routes/layout.templ`

**Issue:** Tailwind CSS loaded from `https://unpkg.com/@tailwindcss/browser@4` with no Subresource Integrity hash. If unpkg.com is compromised, all example apps execute attacker code.

**Mitigation:** Add `integrity="sha384-..." crossorigin="anonymous"` attribute, or bundle CSS statically.

**Severity:** Medium

### 1.5 Medium: Hardcoded `DevMode: true` in Examples

**Files:** `examples/counter/main.go`, `examples/counter-test-prefork/main.go`, `examples/form-remote/main.go`

**Issue:** If a user copies an example as a production starting point without reading carefully, DevMode exposes:
- File watchers
- Hot reload endpoints
- Error overlays with stack traces
- Debug info in remote action errors (`gospa.go:470-472`)

**Mitigation:** Use `os.Getenv("GOSPA_DEV") == "1"` pattern instead of hardcoding.

**Severity:** Medium

---

### 1.6 Medium: Redis No-Auth in Prefork Examples

**Files:** `examples/counter-test-prefork/main.go`, `examples/prefork/main.go`

**Issue:** Redis connection uses `localhost:6379` with no `Password` field. If Redis is exposed on the network, session data and pub/sub messages are readable/writable by anyone.

**Mitigation:** Add `Password: os.Getenv("REDIS_PASSWORD")` and document the requirement.

**Severity:** Medium

## 2. Performance Issues

### 2.1 O(n) Message Insert in form-remote Example

**File:** `examples/form-remote/routes/store.go`
```go
s.messages = append([]Message{msg}, s.messages...)
```

**Impact:** Every new message copies the entire existing slice. O(n) per insert. At 10,000 messages, this becomes a significant CPU bottleneck.

**Fix:** Use a circular buffer or append + reverse for display.
```go
s.messages = append(s.messages, msg)
// Reverse only when reading for pagination
```

**Expected Gain:** O(1) inserts â€” ~50% faster at scale.

---

### 2.2 Unused PostCSS Dependencies

**Files:** `examples/counter/postcss.config.js`, `examples/form-remote/postcss.config.js`, `website/package.json`

**Issue:** `autoprefixer` and `postcss-nested` are included but unnecessary with Tailwind v4 (handles both natively). Dead weight in build pipeline.

**Fix:** Remove from `package.json` and `postcss.config.js`.

**Expected Gain:** Faster CSS builds, smaller node_modules.

---

### 2.3 Eager Search Index Fetch

**File:** `website/main.go`, `static/docs_search_index.json`

**Issue:** The search index (825+ lines) loads on hover of the search button. For users who never search, this is wasted bandwidth.

**Fix:** Defer fetch until the search modal is actually opened and focused.

**Expected Gain:** ~50KB saved per page view for non-searching users.

---

### 2.4 Stale Sitemap

**File:** `website/static/sitemap.xml`

**Issue:** All `lastmod` dates are `2026-03-13`. Gospa SFC sub-pages are missing from sitemap entirely.

**Fix:** Regenerate sitemap on each deploy with current dates and all registered routes.

---

### 2.5 Dead Dependencies in Website

**File:** `website/package.json`

**Issue:** `autoprefixer` and `postcss-nested` are dead weight with Tailwind v4.

**Fix:** Remove both.
