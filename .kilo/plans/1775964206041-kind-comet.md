# Core Web Vitals Optimization Plan — Gospa Framework

## Overview

Comprehensive plan to improve FCP, CLS, LCP, and INP across the Gospa framework. Changes span framework defaults, middleware, rendering pipeline, build tooling, and the reference website.

---

## 1. FCP (First Contentful Paint)

### 1.1 Fix `PreloadHeadersMiddleware` — set Link headers before downstream handlers

**File:** `fiber/middleware.go:310`
**Problem:** `PreloadHeadersMiddleware` calls `c.Next()` first, then sets `Link` headers. By the time headers are set, the browser has already started parsing the HTML body — preload hints arrive too late.
**Fix:** For HTML responses, build and set `Link` headers **before** `c.Next()`. Non-HTML responses skip entirely (unchanged behavior).

### 1.2 Add `HydrationTimeout` default of 2000ms

**File:** `config.go` (in `applyDefaultConfig`)
**Problem:** `HydrationTimeout` defaults to 0. With `HydrationMode: "idle"`, a 0 timeout means "never force hydrate." On low-end devices `requestIdleCallback` may fire seconds after paint, leaving interactive elements dead.
**Fix:** Add `if config.HydrationTimeout == 0 { config.HydrationTimeout = 2000 }`.

### 1.3 Default `FallbackToMicrotask` to true for `IdleCallbackBatchUpdates`

**File:** `config.go` (in `DefaultConfig`)
**Problem:** `NavigationIdleCallbackBatchUpdatesConfig.FallbackToMicrotask` is nil by default. If `requestIdleCallback` is unavailable (Safari < 15.5), batched updates have no fallback path and may run synchronously on the main thread, blocking paint.
**Fix:** Set `FallbackToMicrotask: boolPtr(true)` in `NavigationOptions.IdleCallbackBatchUpdates`.

### 1.4 Defer runtime script execution with `requestIdleCallback`

**File:** `website/routes/root_layout.templ:110-123`
**Problem:** The inline module script does `import(b.gospaRuntimePath).then(...)` immediately. Module scripts are deferred by spec, but the dynamic `import()` still starts parsing during the initial load phase, competing with paint.
**Fix:** Wrap the `import()` in `requestIdleCallback`:
```js
requestIdleCallback(() => {
    import(b.gospaRuntimePath).then(m => {
        m.init({ ... });
    });
}, { timeout: 2000 });
```

### 1.5 Add `font-display: swap` verification to build/doctor

**File:** `cli/doctor.go`
**Problem:** If the user's CSS `@font-face` lacks `font-display: swap`, the browser blocks text rendering until the font downloads (FOIT), directly hurting FCP.
**Fix:** Add a check in the `gospa doctor` command that scans CSS files for `@font-face` blocks missing `font-display`.

## 2. CLS (Cumulative Layout Shift)

### 2.1 Guarantee structural utilities in critical CSS

**File:** `plugin/postcss/postcss.go` — `splitCSS` function
**Problem:** When critical CSS is extracted, the scanner only includes classes found in `criticalContent` templates. If `.hidden`, `.fixed`, or `.absolute` aren't in the scanned templates (or Tailwind's minifier reorders them), they may land in the non-critical CSS. When the non-critical CSS loads async, elements that should be hidden flash visible.
**Fix:** In `splitCSS`, always prepend a safety block to the critical CSS output:
```css
.hidden{display:none!important}.invisible{visibility:hidden}.fixed{position:fixed}.absolute{position:absolute}
```
This is ~80 bytes — negligible impact on the 14KB/32KB budget.

### 2.2 Add `contain: layout style paint` to animated background container

**File:** `website/routes/root_layout.templ:169`
**Problem:** The animated background divs use `blur-[120px]` and `grid-bg`. If the CSS for `.grid-bg` arrives late (async non-critical CSS), the browser may recalculate layout for the entire page.
**Fix:** Add `style="contain: layout style paint;"` to the background container div.

### 2.3 Ensure `criticalContent` in gospa.yaml covers all above-fold components

**File:** `website/gospa.yaml`
**Problem:** The `criticalContent` lists for the `website` and `docs` bundles include `root_layout.templ` and page templates, but the mobile menu overlay (`#mobile-menu`) and search modal (`#search-modal`) are in `root_layout.templ`. Their `.hidden` class should be in critical CSS, but the Tailwind `.hidden` utility may not be captured if the class regex doesn't match the `templ.KV` dynamic class construction.
**Fix:** Add `components/` glob to both bundles' `criticalContent` lists.

### 2.4 Add explicit `width`/`height` on all doc page images

**File:** All doc page `.templ` files in `website/routes/`
**Problem:** Any `<img>` without intrinsic dimensions causes CLS when the image loads.
**Fix:** Audit all doc pages. Add `width` and `height` attributes to every `<img>`.

## 3. LCP (Largest Contentful Paint)

### 3.1 Add HTTP 103 Early Hints support for SSG/ISR cached pages

**File:** `render.go:55-66` (SSG cache hit path) and `render.go:98-114` (ISR cache hit path)
**Problem:** For cached SSG/ISR pages, the HTML is ready instantly but the browser still needs to discover the font, runtime, and CSS. Fiber v3 supports `c.SendStatus(103)`.
**Fix:** Before sending cached HTML, send a `103 Early Hints` response with `Link` headers for the font and runtime script. This requires HTTP/2 or HTTP/3 (which Render's edge supports).

```go
// Before c.Send(entry.html):
c.SendStatus(103)
c.Set("Link", "<...font.woff2>; rel=preload; as=font; crossorigin, <...runtime.js>; rel=modulepreload")
```

### 3.2 Stream cached SSG HTML in 14KB chunks

**File:** `render.go:66` — `return c.Send(entry.html)`
**Problem:** Cached SSG pages send the full HTML buffer at once. For large pages (>14KB), the browser waits for the entire response before it can start painting. The TCP initial congestion window is ~14KB.
**Fix:** Replace `c.Send(entry.html)` with a streaming write in 14KB chunks:
```go
c.Set("Content-Type", "text/html")
c.Response().SetBodyStreamWriter(fasthttp.StreamWriter(func(w *bufio.Writer) {
    for i := 0; i < len(entry.html); i += 14000 {
        end := i + 14000
        if end > len(entry.html) { end = len(entry.html) }
        w.Write(entry.html[i:end])
        w.Flush()
    }
}))
```

### 3.3 Remove duplicate image preload from root_layout.templ

**File:** `website/routes/root_layout.templ:90`
**Problem:** The logo is preloaded twice: once via `<link rel="preload">` in the HTML head (line 90) and again via the `Link` header in `main.go:146-149`. Double-preload wastes a connection slot.
**Fix:** Remove the `<link rel="preload" as="image" href="/static/gospa1-64.webp">` from the HTML. Keep the server-side `Link` header (it arrives earlier in the response).

### 3.4 Register Brotli compression middleware by default

**File:** `gospa.go:562-564`
**Problem:** The default middleware stack uses Fiber's built-in `compress.New(compress.LevelBestSpeed)` which only does gzip. The `fiber/compression.go` file has a full `BrotliGzipMiddleware` that prefers Brotli (better compression ratio → smaller payloads → faster LCP), but it's never registered by default.
**Fix:** Replace the gzip-only compression with `fiber.BrotliGzipMiddleware(fiber.DefaultCompressionConfig())` in `setupMiddleware`.

### 3.5 Add `content-visibility: auto` to below-fold sections

**File:** `website/routes/root_layout.templ:302` (footer)
**Problem:** The footer renders immediately even though it's below the fold. The browser spends paint time on content the user can't see yet.
**Fix:** Add `class="content-visibility-auto"` with `contain-intrinsic-size: 0 500px` to the footer. Create a Tailwind utility or inline style.

## 4. INP (Interaction to Next Paint)

### 4.1 Lazy WebSocket connection

**File:** `embed/runtime.js` (runtime init) and `config.go` (add `WSLazyConnect` config option)
**Problem:** `EnableWebSocket: true` opens a WS connection on every page load. The WS handshake and the runtime's connection management code compete with the main thread during initial load. For content-heavy pages (docs, marketing), real-time state sync isn't needed immediately.
**Fix:** Add `WSLazyConnect bool` to `Config`. When true, the runtime defers the WS connection until the first `sendAction()`, state subscription, or user interaction. Default to `true` for new projects.

### 4.2 Throttle speculative prefetch under main thread pressure

**File:** Client runtime prefetch logic (in `embed/runtime.js` / client source)
**Problem:** Speculative prefetching with `ViewportMargin: 150px` can trigger multiple HTML fetches simultaneously. If the main thread is busy (e.g., user is scrolling or interacting), parsing prefetch responses adds to INP.
**Fix:** In the prefetch logic, check `navigator.scheduling?.isInputPending?.()` before starting a prefetch fetch. If the user is interacting, queue the prefetch for the next idle period.

### 4.3 Disable WebSocket for docs site

**File:** `website/main.go:50`
**Problem:** The docs site has `EnableWebSocket: true` but doesn't use real-time state sync. The WS connection is wasted overhead on every page load.
**Fix:** Set `EnableWebSocket: false` in the website config. Any interactivity (search, mobile menu) works without WS.

### 4.4 Add `touch-action: manipulation` to interactive elements

**File:** `website/routes/root_layout.templ` — all buttons and nav links
**Problem:** Mobile browsers add a 300ms tap delay to buttons and links. This directly impacts INP on mobile.
**Fix:** Add `touch-manipulation` Tailwind class to all `<button>` and `<a>` elements that trigger interactions.

## 5. Build/CLI

### 5.1 Add image optimization to build pipeline

**File:** `cli/build.go`
**Problem:** The site serves multiple image sizes as separate files. No AVIF conversion, no `srcset` generation.
**Fix:** Add an `optimizeImages` step to `BuildWithConfig` that:
- Converts PNG/JPEG to WebP and AVIF (if `plugin/image` is registered)
- Generates `srcset` attributes for `<img>` tags
- Uses the existing `plugin/image/encode.go` infrastructure

### 5.2 Add manifest.json with hashed asset names

**File:** `cli/build.go:591-646` — `generateBuildManifest`
**Problem:** The current manifest stores SHA256 hashes of files, not the mapping from logical names to hashed output names. The `PreloadHeadersMiddleware` uses `BuildManifest` for asset discovery but expects paths like `static/js/runtime-abc123.js`.
**Fix:** After the Bun build completes, walk the output JS directory and build a mapping:
```json
{
  "runtime.js": "static/js/runtime-d5b7dsbq.js",
  "islands.js": "static/js/islands-xyz.js"
}
```
This enables the preload middleware to discover the correct hashed filenames.

### 5.3 Add JS minification for islands bundle in dev

**File:** `cli/dev.go` or `cli/build.go:unifiedClientBuild`
**Problem:** The islands bundle is built with Bun but there's no explicit minification flag for production in all code paths.
**Fix:** Verify `--minify` is always passed in production builds (already done in `cli/build.go:247`). Add a note in docs.

### 5.4 Add SSG warmup endpoint

**File:** `cli/serve.go` or new `cli/warmup.go`
**Problem:** On Render's free tier, the server cold-starts with an empty SSG cache. The first visitor triggers on-demand rendering of every page they visit.
**Fix:** Add a `gospa warmup` command that:
1. Reads all registered routes from the router
2. Makes HTTP requests to each route to populate the SSG cache
3. Can be called as a post-start hook or via Render's health check

## 6. Server/Deployment

### 6.1 Verify HTTP/2 support on Render

**Problem:** If Render's edge only serves HTTP/1.1, the multiple preload links (font, runtime, CSS) queue behind the 6-connection limit. The `PreloadHeadersMiddleware` already caps Link headers based on protocol (6 for HTTP, 12 for HTTPS), but this doesn't change the actual protocol.
**Fix:** Document this limitation. If HTTP/2 isn't available, reduce preload count to 3 and prioritize: font → runtime → critical CSS.

### 6.2 Add `Preconnect` headers for third-party origins

**File:** `website/routes/root_layout.templ:88`
**Problem:** The site preconnects to `/` (self) but not to external origins like `raw.githubusercontent.com` (used for the OG image at line 22).
**Fix:** Add `<link rel="preconnect" href="https://raw.githubusercontent.com" crossorigin>` if the OG image is loaded from GitHub. Better: host the OG image locally.
