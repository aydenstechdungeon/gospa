# Documentation Sync Plan

## Overview

Synchronize documentation across three sources:
1. **`docs/`** - Authoritative Markdown documentation (source of truth)
2. **`website/`** - Browsable documentation website (Templ pages)
3. **Codebase** - Go framework (`gospa.go`, `config.go`, etc.) and client runtime (`client/src/`)

## Current State Analysis

### Documentation Structure

**`docs/` directory** (29 markdown files):
- `01-getting-started/` - 3 files (quick-start, tutorial, scratch-guide)
- `02-core-concepts/` - 5 files (rendering, state, components, islands, routing)
- `03-features/` - 8 files (client-runtime, runtime-api, realtime, security, dev-tools, deployment, gospa-sfc, production-checklist)
- `04-api-reference/` - Split structure: 3 top-level files + subdirectories for configuration (basic, websocket, scaling) and plugins (overview, tailwind)
- `05-advanced/` - 3 files (error-handling, state-pruning, concurrency)
- `06-migration/` - Referenced in docs/README.md but **MISSING** (01-v1-to-v2.md not found)
- `07-troubleshooting/` - 7 files (runtime-init, remote-actions, websocket, hmr, island-hydration, state-sync, build-deployment)
- `08-audits/` - 9 audit reports

**`website/` directory** (60+ Templ page files):
- Routes organized differently from `docs/` structure
- Uses flat organization: `/docs/getstarted/*`, `/docs/routing/*`, `/docs/state-management/*`, etc.
- Contains generated pages under `/docs/gospasfc/*`
- Has its own sidebar navigation in `components/sidebar.templ`

### Key Discrepancies Found

1. **Missing migration doc**: `docs/06-migration/01-v1-to-v2.md` referenced in `docs/README.md` but doesn't exist
2. **Structure mismatch**: `docs/` uses numbered flat structure; `website/` uses nested topic-based structure
3. **Content duplication**: `website/` has its own content that may diverge from `docs/`
4. **API reference split**: `docs/04-api-reference/` has both flat files and subdirectories for configuration/plugins
5. **Client runtime docs**: `docs/03-features/01-client-runtime.md` covers runtime selection; `docs/04-api-reference/01-core-api.md` has extensive client API reference
