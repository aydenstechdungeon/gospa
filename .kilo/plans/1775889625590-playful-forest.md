# GoSPA CLI Enhancement Plan

## Overview

Comprehensive CLI enhancement covering new commands, flags, config file support, and developer experience improvements. The CLI lives in `cli/` directory with each command as a separate `.go` file.

---

## Phase 1: Core Flag Additions (Existing Commands)

### 1.1 `gospa dev` New Flags

**File: `cli/dev.go`**

| Flag | Type | Default | Description |
|------|------|---------|-------------|
| `--port`, `-p` | int | 3000 | Server port (already partially supported) |
| `--host` | string | localhost | Bind address |
| `--open`, `-o` | bool | false | Auto-open browser on start |
| `--routes-dir` | string | ./routes | Routes directory |
| `--no-restart` | bool | false | Disable auto-restart on .go changes |
| `--verbose` | bool | false | Verbose logging output |
| `--timeout` | duration | 30s | Server start timeout before kill |
| `--log-format` | string | text | Log format: text, json |
| `--hmr-port` | int | random | HMR WebSocket port |
| `--proxy` | string | "" | Proxy API requests to backend |
| `--watch-paths` | string | "" | Comma-separated extra watch dirs |
| `--debounce` | duration | 100ms | File change debounce interval |

**Changes:**
- Add `Timeout`, `LogFormat`, `HMRPort`, `Proxy`, `Debounce` fields to `DevConfig` struct
- Update `startServerProcess` to respect timeout
- Add proxy middleware setup when `--proxy` flag set
- Add JSON log formatter
- Wire `--open` flag to `openBrowser()` call

## Phase 2: New Commands

### 2.1 `gospa serve` - Production Server

**New file: `cli/serve.go`**

```
gospa serve                    # Serve from dist/
gospa serve --port 8080        # Custom port
gospa serve ./custom-dist    # Custom directory
gospa serve --https            # HTTPS mode
gospa serve --cert cert.pem    # TLS cert
gospa serve --key key.pem      # TLS key
```

**Config struct:**
```go
type ServeConfig struct {
    Port     int
    Host     string
    Dir      string
    HTTPS    bool
    Cert     string
    Key      string
    Open     bool
    Gzip     bool
    Brotli   bool
    Cache    bool
    Headers  map[string]string
}
```

**Implementation:**
- Serve static files from dist directory
- Optional HTTPS with TLS
- Gzip/Brotli compression
- Cache headers
- Custom headers support

### 2.2 `gospa inspect` - Project Introspection

**New file: `cli/inspect.go`**

```
gospa inspect                  # Full report
gospa inspect --routes         # Route table
gospa inspect --components     # Component tree
gospa inspect --state          # State definitions
gospa inspect --actions        # Remote actions
gospa inspect --islands        # Island components
gospa inspect --middleware     # Middleware chain
gospa inspect --json           # JSON output
```

**Config struct:**
```go
type InspectConfig struct {
    Routes     bool
    Components bool
    State      bool
    Actions    bool
    Islands    bool
    Middleware bool
    JSON       bool
    Output     string  // file path for output
}
```

**Implementation:**
- Parse routes directory and display route table
- Show component hierarchy
- List state definitions from Go files
- List registered remote actions
- Show island components
- Display middleware chain per route

### 2.3 `gospa init` - Initialize in Existing Directory

**New file: `cli/init.go`**

```
gospa init                     # Init in current dir
gospa init --module myapp      # Custom module path
gospa init --template minimal  # With template
```

**Config struct:**
```go
type InitConfig struct {
    Module   string
    Template string
    Force    bool
}
```

**Implementation:**
- Like `create` but for existing directories
- Only creates missing files
- `--force` to overwrite existing files

### 2.4 `gospa build-all` - Multi-Platform Build

**New file: `cli/build_all.go`** (extends existing `BuildAll` function)

```
gospa build-all                        # All default platforms
gospa build-all --targets linux/amd64,linux/arm64,darwin/arm64
gospa build-all --output ./releases
gospa build-all --compress             # Compress binaries
gospa build-all --manifest             # Generate release manifest
```

**Config struct:**
```go
type BuildAllConfig struct {
    Targets   []string
    OutputDir string
    Compress  bool
    Manifest  bool
    Parallel  int
}
```

**Implementation:**
- Parallel builds for multiple platforms
- Optional compression (tar.gz)
- Release manifest generation
- Existing `BuildAll()` function already does this, just add flags

### 2.5 `gospa completion` - Shell Completion

**New file: `cli/completion.go`**

```
gospa completion bash
gospa completion zsh
gospa completion fish
gospa completion powershell
```

**Implementation:**
- Generate shell completion scripts
- Support bash, zsh, fish, powershell
- Use cobra's built-in completion generation (if migrating) or manual templates

### 2.6 `gospa add` / `gospa remove` - Plugin Management

**New file: `cli/plugins.go`**

```
gospa add tailwind                 # Add official plugin
gospa add github.com/user/plugin   # Add external plugin
gospa remove tailwind              # Remove plugin
gospa list                         # List installed plugins
gospa update                       # Update all plugins
gospa update tailwind              # Update specific plugin
```

**Config struct:**
```go
type PluginConfig struct {
    Name    string
    Source  string  // official, github, local
    Version string
    Force   bool
}
```

**Implementation:**
- Download plugins to `~/.gospa/plugins/`
- Update `gospa.json` config
- Version management
- Dependency resolution

### 2.7 `gospa state` - State Management CLI

**New file: `cli/state.go`** (extends existing prune.go)

## Phase 3: Configuration File Support

### 3.1 Config File Format

**New file: `cli/config.go`**

Supported files (in order of precedence):
1. `gospa.config.yaml`
2. `gospa.config.json`
3. `gospa.config.toml`
4. `.gospa.yaml`
5. `.gospa.json`

**Schema:**
```yaml
# gospa.config.yaml

project:
  name: myapp
  module: github.com/user/myapp

dev:
  port: 3000
  host: localhost
  open: true
  routes_dir: ./routes
  watch_paths:
    - ./components
    - ./lib
  proxy: http://localhost:8081
  hmr_port: 0  # auto
  debounce: 100ms
  timeout: 30s

build:
  output: dist
  minify: true
  compress: true
  sourcemap: false
  cgo: false
  env: production
  assets_dir: static
  targets:
    - platform: linux
      arch: amd64
    - platform: darwin
      arch: arm64

generate:
  output: ./generated
  type: island
  strict: false

serve:
  port: 8080
  gzip: true
  brotli: true
  cache: true
  headers:
    X-Frame-Options: DENY
    X-Content-Type-Options: nosniff

plugins:
  - name: tailwind
    version: latest
  - name: postcss
    version: latest
```

### 3.2 Config Loading

**Implementation:**
- Use `gopkg.in/yaml.v3` for YAML parsing
- Merge config file + CLI flags (flags take precedence)
- Validate config on load
- Provide `gospa config show` to display effective config

### 3.3 `gospa config` Command

**New file: `cli/config_cmd.go`**

```
gospa config show                # Show effective config
gospa config show --json         # JSON output
gospa config init                # Create default config file
gospa config set dev.port 8080   # Set a value
gospa config get dev.port        # Get a value
```

## Phase 4: Developer Experience

### 4.1 Colored Output & Progress

**Enhance: `cli/output.go`**

- Progress bars for long operations (build, generate)
- Spinner for dev server start
- Better error messages with suggestions
- Consistent color scheme across all commands

### 4.2 Environment Variables

All flags should be overridable via environment variables:

```
GOSPA_DEV_PORT=8080 gospa dev
GOSPA_BUILD_OUTPUT=./releases gospa build
GOSPA_GENERATE_OUTPUT=./types gospa generate
```

**Implementation:**
- Add `getEnvOrFlag(envVar, defaultValue)` helper
- Document all env vars in help text

### 4.3 Help & Documentation

- Better `--help` output with examples
- `gospa help <command>` for detailed help
- Interactive `gospa` (no args) shows menu

### 4.4 Debug Mode

```
GOSPA_DEBUG=1 gospa dev    # Enable debug logging
gospa dev --debug          # Same via flag
```

**Implementation:**
- Debug logger in `cli/output.go`
- Log file output: `gospa dev --log-file gospa.log`

## Phase 5: Advanced Features

### 5.1 `gospa preview` - Preview Deployments

**New file: `cli/preview.go`**

```
gospa preview                    # Build + serve locally
gospa preview --port 8080        # Custom port
gospa preview --build-flags      # Pass build flags
```

**Implementation:**
- Combines build + serve
- Useful for testing production builds locally

### 5.2 `gospa bundle` - Bundle Analysis

**New file: `cli/bundle.go`**

```
gospa bundle analyze             # Analyze bundle size
gospa bundle analyze --json      # JSON output
gospa bundle visualize           # Generate treemap
```

**Implementation:**
- Analyze JS bundle sizes
- Show largest modules
- Optional treemap visualization (HTML output)

### 5.3 `gospa test` - Test Runner

**New file: `cli/test.go`**

```
gospa test                       # Run all tests
gospa test --unit                # Unit tests only
gospa test --e2e                 # E2E tests only
gospa test --coverage            # With coverage
```

**Implementation:**
- Wrapper around `go test`
- E2E test support with headless browser
- Coverage reports

### 5.4 `gospa deploy` - Deployment Helpers

**New file: `cli/deploy.go`**

```
gospa deploy docker              # Build Docker image
gospa deploy docker --tag v1.0   # Custom tag
gospa deploy docker --push       # Push to registry
gospa deploy binary              # Just build binary
gospa deploy static              # Static site export
```

**Implementation:**
- Docker build with optimized multi-stage Dockerfile
- Binary deployment
- Static site export (SSG mode)
