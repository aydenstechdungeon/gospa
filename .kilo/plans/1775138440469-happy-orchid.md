## 1. Security Vulnerabilities & Exploits

### 1.1 CRITICAL Vulnerabilities

#### 1.1.1 No JWT Secret Minimum Length Validation
**File:** `plugin/auth/auth.go:50`, `plugin/auth/auth.go:377-421`
**OWASP:** A02:2021 — Cryptographic Failures

The `JWTSecret` field accepts any string with no minimum length enforcement. In production, it only checks existence, not strength:

```go
jwtSecret := os.Getenv("JWT_SECRET")
if jwtSecret == "" && isProductionRuntime() {
    panic("JWT_SECRET environment variable is not set.")
}
// No length check — "abc" passes
```

**PoC:** `JWT_SECRET=abc` → HMAC-SHA256 with 24-bit key → brute-forceable in seconds.

**Mitigation:** Add minimum length validation:
```go
if len(jwtSecret) < 32 {
    panic("JWT_SECRET must be at least 32 characters for HS256 security")
}
```

#### 1.1.2 Default CSP Allows `script-src 'unsafe-inline'`
**File:** `fiber/middleware.go:393`
**OWASP:** A03:2021 — Injection (XSS)

The default CSP permits inline script execution, completely negating XSS protection:

```go
const DefaultContentSecurityPolicy = "... script-src 'self' 'unsafe-inline'; ..."
```

**Impact:** Any HTML injection becomes full XSS. The framework has `StrictContentSecurityPolicy` available but doesn't use it by default.

**Mitigation:** Remove `'unsafe-inline'` from `script-src`. Use nonce-based CSP for generated inline scripts. The framework already generates inline scripts (`render.go:274-294`) which would need nonce attributes.

#### 1.1.3 WebSocket Session Token Replayable via Header
**File:** `fiber/websocket.go:1604`
**OWASP:** A01:2021 — Broken Access Control

Session tokens are accepted via `X-Session-Token` HTTP header without IP binding or rotation:

```go
func StateSyncHandler(_ *WSHub) fiberpkg.Handler {
    return func(c fiberpkg.Ctx) error {
        sessionToken := c.Get("X-Session-Token")
```

With a 24-hour TTL and no token rotation, a stolen token grants prolonged access.

**Mitigation:** Bind sessions to client IP/fingerprint, implement token rotation on use, enforce TLS-only.

#### 1.1.4 Plugin Uninstall Path Traversal
**File:** `plugin/loader.go:461`
**OWASP:** A01:2021 — Broken Access Control

```go
func UninstallPlugin(name string) error {
    owner, repo, _, err := ParsePluginRef(name)
    pluginPath := filepath.Join(PluginCacheDir, owner, repo)
    return os.RemoveAll(pluginPath) // Deletes arbitrary directory
}
```

`ParsePluginRef` only checks for empty strings and `/` presence. A crafted name like `../../etc` could delete files outside the cache.

**Mitigation:** Add `filepath.Clean` and verify resolved path stays within `PluginCacheDir`:
```go
pluginPath := filepath.Clean(filepath.Join(PluginCacheDir, owner, repo))
absPath, _ := filepath.Abs(pluginPath)
absCacheDir, _ := filepath.Abs(PluginCacheDir)
if !strings.HasPrefix(absPath, absCacheDir+string(filepath.Separator)) {
    return fmt.Errorf("invalid plugin path")
}
```

#### 1.1.5 `Rune.Update()` TOCTOU Race Condition
**File:** `state/rune.go:235-262`
**Category:** Concurrency / Data Loss

```go
func (r *Rune[T]) Update(fn func(T) T) {
    r.mu.Lock()
    oldValue := r.value
    r.mu.Unlock()       // UNLOCK — fn runs outside lock

    newValue := fn(oldValue)  // Another goroutine can modify r.value here

    r.mu.Lock()
    if equal(r.value, newValue) {  // r.value may have changed!
```

**Impact:** Lost updates when two goroutines call `Update` concurrently. One goroutine's update is silently discarded.

**Mitigation:** Hold the lock during the entire read-modify-write, or use compare-and-swap.

#### 1.1.6 `defaultApp` Global Variable Race
**File:** `gospa.go:149-151`
**Category:** Concurrency

```go
if defaultApp == nil {
    defaultApp = app
}
```

No synchronization on the global variable. Concurrent `New()` calls cause a data race.

**Mitigation:** Use `sync.Once` or atomic operations.

### 1.2 HIGH Vulnerabilities

#### 1.2.1 Template Injection via Unescaped Expression Content
**File:** `compiler/compiler.go:339`
**Risk:** Requires untrusted `.gospa` files

Expression content is injected verbatim into generated templ code:

```go
case *sfc.ExpressionNode:
    sb.WriteString("{ ")
    sb.WriteString(n.Content)  // UNESCAPED
    sb.WriteString(" }")
```

**PoC:** `<div>{ os.RemoveAll("/") }</div>` generates executable Go code in the template.

**Mitigation:** Make `SafeMode: true` the default, or add a warning when compiling from untrusted sources.

#### 1.2.2 Dev Panel DOM XSS
**File:** `fiber/dev.go:530-534`
**Risk:** Dev-mode only

```javascript
div.innerHTML = '<span class="log-key">' + entry.key + '</span>' +
    '<span class="log-value" title="' + entry.newValue + '">...'
```

State keys/values injected into `innerHTML` without sanitization.

**PoC:** WebSocket state update with key `"><script>alert(1)</script>` executes JS.

**Mitigation:** Use `textContent` instead of `innerHTML`, or use DOMPurify.

#### 1.2.3 XSS via Islands Path Injection
**File:** `render.go:303`

```go
_, _ = fmt.Fprintf(w, `<script src="/%s"></script>`, islandsPath)
```

`IslandsBundlePath` injected into HTML without escaping. If attacker can influence config, they can inject HTML/JS.

**Mitigation:** `html.EscapeString(islandsPath)`

#### 1.2.4 No OTP Brute-Force Protection
**File:** `plugin/auth/auth.go:1385-1398`
**OWASP:** A07:2021 — Identification and Authentication Failures

With 6-digit OTP codes (1M possibilities) and a 30-second window, an attacker has ~3M valid codes per window. No rate limiting exists.

**Mitigation:** Per-user and per-IP rate limiting. Lock after 5-10 failed attempts with exponential backoff.

#### 1.2.5 Connection ID Truncated to 32 Bits
**File:** `fiber/websocket.go:1181`

```go
connID := "conn_" + generateSecureToken()[:8] // Only 8 hex chars = 32 bits
```

2^32 ≈ 4.3 billion possible IDs. Brute-forceable and collision-prone in high-traffic environments.

**Mitigation:** Use at least 16 hex characters (64 bits) or the full token.

#### 1.2.6 SSE Unsubscribe Missing Default Authorization
**File:** `fiber/sse.go:407-441`
**OWASP:** A01:2021 — Broken Access Control

Unlike `SSESubscribeHandler` (which validates session identity), `SSEUnsubscribeHandler` relies solely on the optional `authorizeSubscribe` hook. If not configured, any client can unsubscribe any other client.

**Mitigation:** Apply same session identity verification as subscribe handler.

#### 1.2.7 Redis TLS Not Enforced
**File:** `store/redis/redis.go:20-25`
**OWASP:** A02:2021 — Cryptographic Failures

The Redis client is accepted as-is with no TLS validation. Session tokens transit in cleartext if not configured externally.

**Mitigation:** Add TLS validation helper or panic if `client.Options().TLSConfig == nil` in production.

#### 1.2.8 Backup Code Hash Lacks Key Stretching
**File:** `plugin/auth/auth.go:1439-1448`

HMAC-SHA256 with random salt, but no key stretching (bcrypt/argon2). Backup codes are 48 bits of entropy → brute-forceable with fast hashing.

**Mitigation:** Use `argon2id` or `bcrypt` with appropriate work factor.

## 3. Bugs & Logic Errors

### 3.1 HIGH Bugs

#### 3.1.1 `MinimalConfig()` WebSocket Re-enabled by `applyDefaultConfig`
**File:** `gospa.go:160-162`

```go
if !config.EnableWebSocket && config.WebSocketPath == "" {
    config.EnableWebSocket = true
}
```

`MinimalConfig()` sets `EnableWebSocket = false` and `WebSocketPath = ""`. Then `applyDefaultConfig` sees `!false && "" == ""` → `true`, so it re-enables WebSocket — directly contradicting the user's intent.

**Repro:**
```go
config := MinimalConfig() // EnableWebSocket = false
app := New(config)        // applyDefaultConfig re-enables WebSocket!
// app.Config.EnableWebSocket is now true
```

**Fix:** Remove this condition or change logic to only set default path without re-enabling.

#### 3.1.2 `-1` Unlimited SSG/PPR Cache Broken
**Files:** `render_ssg.go:17-19`, `render_ppr.go:22-25`

```go
maxEntries := a.Config.SSGCacheMaxEntries
if maxEntries <= 0 {
    maxEntries = 500
}
```

When `SSGCacheMaxEntries = -1` (unlimited), this code resets it to 500. The unlimited cache feature is broken.

**Fix:**
```go
if maxEntries == 0 { maxEntries = 500 }
if maxEntries < 0 { /* unlimited — skip eviction */ }
```

#### 3.1.3 SSR Streaming Path Missing Panic Recovery
**File:** `render.go:262-326`

The PPR streaming path has panic recovery, but the fallback SSR streaming path does NOT. A panic in `content.Render()` crashes the server.

**Fix:** Add `defer recover()` in the StreamWriter.
