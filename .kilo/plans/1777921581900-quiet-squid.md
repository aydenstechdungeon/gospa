## Phase 1: HIGH Priority (Security/Correctness)

### Issue #1 — CSRF Token Exposed to JavaScript
- **File**: `fiber/middleware.go:133-138`
- **Fix**: Remove `window.__GOSPA_CSRF_TOKEN__` injection from StateMiddleware. The cookie is already HttpOnly, so JS can't read it. The CSRFTokenMiddleware reads the token from `X-CSRF-Token` header (line 391), which JS should NOT be able to set. Remove the injection and change CSRFTokenMiddleware to read the token from the cookie directly via double-submit pattern where the cookie value is compared to the header — but since the cookie is HttpOnly, forms must use the `_csrf` field fallback only. Remove JS global injection; keep `_csrf` form field support.
- **Impact**: CSRFTokenMiddleware already reads `X-CSRF-Token` header and `_csrf` form value. After fix: only `_csrf` form field and cookie-to-header comparison via server-side template injection of hidden form fields (existing behavior).

### Issue #7 — SSG Cache Serves Stale Nonce
- **Files**: `render.go:67`, `render.go:348`, `render.go:148`, `render_utils.go:347-348`
- **Fix**: Nonces should NOT be embedded in cached HTML at all. Instead, use a middleware that rewrites `nonce="{nonce}"` placeholders in cached HTML just before serving. The current `replaceNonces` approach is broken because it replaces a placeholder with one request's nonce. Fix: the CSP nonce must be injected at request time into the HTML that's served, not stored in cache. Use a streaming `io.Reader` wrapper that does `bytes.ReplaceAll` on the fly, or keep the placeholder approach but ensure the middleware always runs AFTER cache lookup (move nonce injection out of render and into a post-cache middleware).

### Issue #8 — Batch Panic Recovery Drops Mutations
- **File**: `state/batch.go:160-178`
- **Fix**: Document the behavior (panic = dirty notifiers lost) and ensure the `panicked` flag also decrements `activeSyncBatchCount` correctly. The current defer at 161-166 handles cleanup but `flushBatch` is skipped on panic — this is by design since partial state should not be flushed. Add a comment documenting this trade-off.

### Issue #13 — deepEqual No Cycle Detection
- **Files**: `fiber/websocket.go:1182-1309`, `state/serialize.go:346-436`
- **Fix**: Add a depth limit of 64 to both `deepEqual` functions. When depth exceeds limit, return `false` (values are considered different rather than risking stack overflow).

### Issue #19 — Predictable randomString Fallback
- **File**: `fiber/middleware.go:790-817`
- **Fix**: When `crypto/rand` fails, return an error or use a fallback that's not time-based. Since `generateComponentID` uses `randomString` and is called in many places, change `randomString` to return `([]byte, error)` and propagate errors up, or use a secondary crypto source.

## Phase 2: MEDIUM Priority (Security Hardening)

### Issue #2 — WS Init Sends Token in Body
- **Files**: `client/src/websocket.ts:437-438`, `client/src/websocket.ts:738-745`
- **Fix**: Remove `sessionToken` from WS init message in `connect()`. The server already reads the session from the `gospa_session` cookie sent automatically with the WebSocket handshake. The `SendInitWithSession` at `websocket.go:1543` sends the token to the client — this is OK for reconnection but the client should NOT send it back.

### Issue #3 — HMR WebSocket Unauthenticated
- **File**: `fiber/hmr.go:522-527`
- **Fix**: Add origin validation to `HMREndpoint()`. Check `Origin` header against `localhost` / `127.0.0.1` in dev mode. Reject connections from unexpected origins.

### Issue #4 — safeIslandTSPath Symlink Bypass
- **File**: `gospa.go:59`
- **Fix**: Add `filepath.EvalSymlinks` call on `targetAbs` before the prefix check to resolve symlinks.

### Issue #12 — isInMemoryStorage String Matching
- **File**: `gospa.go:398-407`
- **Fix**: Use type assertion against a known interface or check the type name more precisely using `reflect.TypeOf(storage).Elem().Name()`.

### Issue #14 — matchRouteSegments Clones Params
- **File**: `routing/auto.go:551-560`
- **Fix**: Use save/restore semantics — mutate a single params map in place and undo changes on backtracking, instead of cloning on every step.

### Issue #20 — HMRFileWatcher.Stop Race
- **File**: `fiber/hmr.go:133-145`
- **Fix**: Use `sync.Once` for stop channel to prevent double-close panics on repeated Stop() calls.

### Issue #21 — Rate Limiter Goroutine Leak
- **File**: `fiber/websocket.go:220-240`
- **Fix**: Register shutdown hooks for global rate limiters. Add `Close()` calls in `App.Shutdown()`.
