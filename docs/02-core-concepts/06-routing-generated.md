# Generated Routing & Types

Running `gospa generate` produces several critical artifacts in your project that bridge the gap between Go and TypeScript. Understanding these files is key to effective development with GoSPA.

## 1. Go Route Registry
**Location:** `routes/generated_routes.go`

This file is automatically generated whenever you add, remove, or rename a `.templ` or `.gospa` file in your `routes/` directory.

### What it does:
- Scans your `routes/` directory for page components.
- Automatically calls `routing.Register()` for each route.
- Ensures that the server knows which Go function to call when a specific URL path is requested.

> [!IMPORTANT]
> Do not edit this file manually. It is overwritten every time you run `gospa generate`.

---

## 2. TypeScript Routes
**Location:** `generated/routes.ts`

This file contains the client-side equivalent of your route registry.

### What it does:
- Exports a `routes` array containing metadata for all paths (params, file locations, dynamic segments).
- Provides the `getRoute(path)` helper for finding route metadata by URL.
- Provides the `buildPath(route, params)` helper for constructing type-safe URLs.

### Example Usage:
```typescript
import { routes, buildPath } from "../generated/routes";

const userRoute = routes.find(r => r.path === "/users/[id]");
const url = buildPath(userRoute, { id: "123" }); // "/users/123"
```

---

## 3. TypeScript Types
**Location:** `generated/types.ts`

This file bridges your Go data structures to the TypeScript runtime.

### What it does:
- Converts any Go struct marked with the `// gospa:state` comment into a TypeScript interface.
- Automatically maps Go types (`int`, `string`, `bool`, slices, maps) to their TypeScript equivalents.
- Enables full IntelliSense and type-checking for your reactive state in `.gospa` files and client-side scripts.

### Example:
If you have this in Go:
```go
// gospa:state
type UserProfile struct {
    Name  string `json:"name"`
    Level int    `json:"level"`
}
```

GoSPA generates this in `types.ts`:
```typescript
export interface UserProfile {
    name: string;
    level: number;
}
```

---

## 4. Compiled SFC Artifacts
When you have `.gospa` files, the generator creates two additional items:

1. **`generated_<name>.templ`**: A companion Templ file that contains the server-side markup and Go logic extracted from your SFC.
2. **`<output>/<hash>.ts`**: A compiled TypeScript hydration island (only for `island` types) that is bundled and sent to the client.
